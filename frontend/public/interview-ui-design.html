I'd use a token bucket per API key stored in Redis with Lua atomicity. For failure resilience, I'd design fail-open with a circuit breaker — temporarily disabling rate limiting if Redis is down rather than blocking all traffic. On the client side, I'd return 429 with a Retry-After header. Trade-off: stricter limits increase latency slightly due to the Redis round-trip per request.
